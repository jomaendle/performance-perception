About this project

User Perceived Performance - Bachelor Theses

Developed by Johannes Maendle
Supervised by Carsten Schlipf
May - Sep, 2019
Boeblingen, Germany

Goal

The goal of this user test is to evaluate methods and techniques which improve user perceived performance in web applications.

Requirements

The application should be solely based on static HTML-files, plain-CSS and Vanilla JavaScript in order to make the application easly accessible to all potential testers. The usage of third-party frameworks could lead to set up overhead which should be avoided. Furthermore, this application should be accessible within the Micro Focus-Intranet. Therefore this site is being hosted on the internal GitHub Enterprise to achieve this. Additionally, GitHub Pages supports serving only static HTML pages. However, this requirement has already been regarded with respect to the first point.

Set Up

A simple web shop selling certain products is used to execute this test operation. The web shop is delievered in different versions, each one containing different approaches in order to affect the perceived performance of the application by the user.

  1. No Loading Enhancement The site operates with set loading times but without any visual methods to improve the perceived performance
  2. Loaders and Spinners Like the previous, this site operates with set loading times. However it contains simple loadig animations, such as loaders and spinners, including the Micro Focus-loading spinner.
  3. Skeleton Screens This site operates with set loading times including skeleton screens. While loading, it shows shapes or placeholder of the content that is to be loaded. Additionally, it introduces an animation for the check-out process, which is the longest procedure during the process.
  4. Combination of Methods As a final test page, this version contains various techniques based on defined criteria on when to use which improvement. This pages includes Skeleton Screens, Spinners but also Page Transitions realized through sliders.

Execution

For test execution, each participant receives the task to buy an iPhone X in the provided web shop. During the test, several anonymous, technical data is collected in order to determine the user satisfaction and perceived performance. However, perceived performance cannot be identified by tehcnical measures only, since perception is subjective and therefore different from person to person. This means that after each test execution every participant is subject to answer a predefined questionnaire to evaluate the effectiveness of the used methods.
